# 暗色配色原理 · Color Principles

> 解释为什么这套暗色搭配在长时间使用下不刺眼、视觉舒适,并给出切换到"天空蓝"主色的迁移方案。

---

## 1. 为什么这套暗色舒适?三条核心原理

### 原理 1:真黑底 + 微暖近白文本(不是"深蓝灰 + 纯白")

| 配对 | 体感 | 物理原因 |
|---|---|---|
| 背景 `#000000` | OLED 像素完全关闭,无屏幕泛光 | 黑色无光通量,不产生屏幕辉光 |
| 背景 `#0f172a`(常见深蓝灰) | 屏幕持续发蓝光 | OLED 微亮像素,长时间用眼会"蓝光疲劳" |
| 文本 `#ededef`(微暖近白) | 在黑底上柔和、不刺眼 | 略偏暖的近白模拟"印刷品"的反射光 |
| 文本 `#ffffff`(纯白) | 在黑底上产生光晕、眼疲劳 | 纯白与纯黑的对比度过载(21:1),瞳孔无法稳定调节 |

**结论:不要用纯黑配纯白。** 纯黑配偏暖近白 (`#000` + `#ededef`) 才是 OLED 屏的舒适解。

---

### 原理 2:四级灰阶 elevation,**用亮度差替代阴影**

```
Surface 0  #000000  ← Body 底
Surface 1  #0a0a0a  ← Input 内底 / 二级容器
Surface 2  #111111  ← 标准卡片(最常用)
Surface 3  #181818  ← Hover 态
Surface 4  #1c1c1f  ← Dialog / Popover 浮层
```

- 每级亮度仅相差 OKLCH `L ≈ 0.04`,**肉眼勉强可分辨**,但足以区分层级。
- `rgba(0,0,0,*)` 阴影在 `#000` 底上**完全不可见**,只浪费 GPU 像素。所以**静态卡片不加 shadow**,层级靠"再亮一格灰"传达。
- 浮层(Dialog / Popover)才使用深阴影 `rgba(0,0,0,0.55)`,跟它叠加在 Surface 2/3 上时形成可察觉的"漂浮感"。

**结论:暗色 elevation 是"亮度的事",不是"阴影的事"。**

---

### 原理 3:单一强调色,出现频率受控

- 整套界面只有**一种**饱和色:emerald `#22c55e`(或后续切换的任何单色)。
- 一屏内目标出现 **3–6 次**(CTA 按钮 1–2 次 + 链接 2–3 次 + 状态徽章 1–2 次)。
- 品牌色 = CTA 色 = 链接色 = 成功状态色 —— **一色四用**。
- 大脑不需要在多种"语义颜色"之间切换,视觉焦点自然落到唯一的彩色块上。

**反例:** 同时用蓝色作链接、绿色作成功、紫色作品牌、橙色作 CTA → 信息密度看起来高,实际是"色彩噪音",注意力被分散。

## 3. 数值规则(WCAG / 对比度)

| 配对 | 对比度 | WCAG |
|---|---|---|
| `#ededef` on `#000` | 18.5:1 | AAA(正文标准) |
| `#ededef` on `#111` | 15.7:1 | AAA |
| `#a1a1aa` on `#000` | 9.5:1 | AAA(副文本) |
| `#8a8a97` on `#000` | 6.4:1 | AA Large |
| `#22c55e` on `#000` | 8.7:1 | AAA(链接 / 状态) |
| **`#000` on `#22c55e`** | **5.6:1** | **AA Normal(CTA 黑字)** |
| `#fff` on `#22c55e` | 2.5:1 | 不通过(**所以不用白字**) |

**记住一条:** 任何高饱和度浅色块(emerald / sky / amber)作为按钮底色时,**用黑字**;只有深色块(blue-600+ / red-600+)才可以用白字。

## 4. 切换到"天空蓝 / 夏天感"主色

### 颜色选型

"天空蓝 + 夏天感"对应的色相是 `sky-500` 附近,L 在 0.7 左右才有"清爽 / 通透"感。

| token | 当前(Emerald) | 切换后(Sky · 推荐) | 备选(Indigo · 偏冷) |
|---|---|---|---|
| `primary` | `#22c55e` | `#0ea5e9` | `#6366f1` |
| `primary-soft` | `#4ade80` | `#38bdf8` | `#818cf8` |
| `primary-deep` | `#16a34a` | `#0284c7` | `#4f46e5` |
| `primary-press` | `#15803d` | `#0369a1` | `#4338ca` |
| `primary-glow` | `rgba(34,197,94,0.08)` | `rgba(14,165,233,0.08)` | `rgba(99,102,241,0.08)` |
| `primary-subdued` | `rgba(34,197,94,0.12)` | `rgba(14,165,233,0.12)` | `rgba(99,102,241,0.12)` |
| **On Primary(CTA 文字)** | `#000` 黑字 | **`#000` 黑字保留** | `#fff` 白字(indigo 需切白) |
| Ring focus halo | `rgba(34,197,94,0.35)` | `rgba(14,165,233,0.4)` | `rgba(99,102,241,0.4)` |
| Success 状态色 | `#22c55e`(同 primary) | `#22c55e`(**保留绿,不要染蓝**) | `#22c55e`(同) |

**关键判断:**
- 推荐 **Sky 方案(`#0ea5e9`)** —— "夏天感"最匹配。L 约 0.71,跟 emerald 同亮度,在 `#000` 底上视觉重量一致,CTA 上仍可用黑字。
- Indigo 偏冷紫,更"科技 / 严肃"少了夏天感,且亮度偏低(L 0.55),CTA 上需切白字,且与暗底对比拉得不够。

---

### 落地步骤(2 步切换)

#### Step 1 · 修改 `web/src/assets/main.css`

```css
/* 把 :root 和 .dark 中,所有 hue 142.5 的 oklch 改为 hue 230 */

:root {
  /* 其余不变 */
  --primary: oklch(0.71 0.14 230);            /* #0ea5e9 sky-500 */
  --primary-foreground: oklch(0 0 0);          /* 仍是黑字 */
  --primary-soft: oklch(0.78 0.12 230);        /* #38bdf8 sky-400 */
  --ring: oklch(0.71 0.14 230 / 40%);
  --chart-1: oklch(0.71 0.14 230);             /* 同步 chart 主色 */
  --sidebar-primary: oklch(0.71 0.14 230);
  --sidebar-primary-foreground: oklch(0 0 0);
  --sidebar-ring: oklch(0.71 0.14 230 / 40%);
}

.dark {
  /* 其余不变 */
  --primary: oklch(0.71 0.14 230);             /* #0ea5e9 sky-500 */
  --primary-foreground: oklch(0 0 0);
  --primary-soft: oklch(0.8 0.13 230);         /* 暗底亮一档 #5ac6f5 */
  --ring: oklch(0.71 0.14 230 / 35%);
  --chart-1: oklch(0.78 0.13 230);
  --sidebar-primary: oklch(0.71 0.14 230);
  --sidebar-primary-foreground: oklch(0 0 0);
  --sidebar-ring: oklch(0.71 0.14 230 / 35%);
}
```

**只动 4 个变量** × 2 个主题 = 8 行修改,**其余不变**。这就是"语义化 token"的好处。

#### Step 2 · 单独保留语义"成功"色

`Success` / 状态徽章 / 在线指示 dot **仍用 `#22c55e` 绿色**,**不要染成蓝色**。蓝色无法表达"成功"语义,会让用户分不清"在线"与"提示信息"。

如果项目里目前 `success` 直接复用了 `primary`(emerald 一色四用),换主色后必须把 `success` 拆出来,单独定义:

```css
:root, .dark {
  --success: oklch(0.72 0.18 142.5);  /* 保留 emerald 仅作语义 */
  --success-soft: oklch(0.72 0.18 142.5 / 12%);
}
```

然后状态指示(`StatusCard` dot、bridge 在线徽章)用 `--success` 而不是 `--primary`。

### 视觉对照(切换后预期效果)

| 元素 | 切换前(Emerald) | 切换后(Sky) |
|---|---|---|
| 主 CTA 按钮 | 绿底黑字 pill | **蓝底黑字 pill,更清爽** |
| 侧栏 active 项 | 绿色填充 | **蓝色填充,更"科技感"** |
| 内联链接 | 浅绿文字 | **浅蓝文字,像超链接的传统期待** |
| Focus ring | 绿色 halo | **蓝色 halo** |
| 在线状态 dot | 绿色 | **绿色不变**(语义色独立) |
| Hero 装饰 glow | emerald 晕染 | **sky 晕染,夏日蓝天感** |
| Hover 卡片底 | `#181818` 不变 | `#181818` 不变 |

---

## 5. 一句话总结

> 暗色之所以舒适,是因为**纯黑底降低发光、近白文本避免眩光、四级灰阶替代阴影、单一强调色控制视觉密度**。换主色时只动 `primary` 衍生色和 `ring`、`chart-1`、`sidebar-primary` 四个 token,把"成功"语义独立出来,其他全套规范保持不变 —— 这就是设计系统真正的价值。
